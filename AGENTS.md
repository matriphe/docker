# AGENTS.md - AI Agent Guidelines

This file provides guidance for AI agents working in this repository. It establishes consistent conventions for commit messages, pull requests, and contribution tracking to ensure proper attribution and traceability of AI-assisted work.

---

## Commit Message Prefixes

**Always** use the appropriate prefix based on the AI agent being used.

### Agent Prefix Reference Table

| Agent             | Prefix                |
|-------------------|-----------------------|
| Claude (Code)     | `[CLAUDE_AI]`         |
| Gemini-CLI        | `[GEMINI_AI]`         |
| Cursor            | `[CURSOR_AI]`         |

| OpenAI Codex      | `[OPENAI_CODEX_AI]`   |

| Opencode          | `[OPENCODE_AI]`       |

| Openclaw          | `[OPENCLAW_AI]`       |

---

### Fallback Patterns

- **Recognized but unmapped agents**: Use `[AGENT_NAME_AI]` where AGENT_NAME is the uppercase version of the agent name
- **Completely unrecognized agents**: Use generic `[AI]` prefix

---

### Model Information Requirements

**Required:** Agents must include the LLM model name in the commit message body for traceability.

For OpenAI Codex, use this model naming pattern in commit bodies:
- `GPT-5.x-Codex (Medium)`
- Replace `Medium` with the active mode when applicable (for example `Low` or `High`).

**Common Model Examples:**
- SWE-1.5, Gemini Flash 2.5, Gemini Pro
- Claude Sonnet-4.5, Claude Opus-4.6
- DeepseekR1

### Format Examples

```bash
[WINDSURF_AI] Fix nginx configuration template

Optional details.

Co-authored-by: Windsurf <noreply@windsurf.ai>
Model: Claude Sonnet-4.5 (Cascade)
```

```bash
[OPENAI_CODEX_AI] Optimize Docker layer caching

Optional details.

Co-authored-by: Codex <codex@openai.com>
Model: GPT-5.3-Codex (Medium)
```

**Required Co-Author Trailer:** Add the agent identity using a `Co-authored-by:` trailer in every AI-generated commit message.

---

### Troubleshooting

**Q: What if the agent I'm using isn't listed?**
A: Use the fallback pattern: `[AGENT_NAME_AI]` where AGENT_NAME is the uppercase version of the agent name. If completely unrecognized, use `[AI]`.

**Q: What if the model I'm using is unknown or not listed?**
A: Include the model name as best you can identify it. If unknown, omit the model line but always include the agent prefix and co-author trailer.

## Pull Request

- **Always** write a short description of the changes in the title.
- Add the more detailed changes on the description. What issues are you solving, and if there is a issue ticket, link it as well.
- **Always** identify your self and the model you use in the end of the description

### Checklist

- [ ] Short, descriptive PR title
- [ ] Detailed description explaining the changes and linked issues
- [ ] Self-identification and model information at the end

## Rationale

### Traceability
Understanding which AI agent made specific changes helps track the origin of contributions and maintain a clear history of automated work.

### Accountability
Tracking performance and patterns of different agents enables evaluation of their effectiveness and identification of improvements.

### Debugging
When issues arise, knowing the context and capabilities behind changes helps understand why certain decisions were made and identify root causes.

### Documentation
Maintaining clear history of automated contributions ensures future maintainers can understand the provenance and reasoning behind code changes.
